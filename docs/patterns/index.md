# 实用 Patterns

> 常见问题模型与可复用实现模式

本目录聚焦“怎么写得更稳”：审计日志、权限门禁、任务编排、会话模式等。

## 目录

- [第10章：实战模式](/patterns/10-patterns)

## 推荐阅读顺序

1. 先读 [10-patterns](/patterns/10-patterns) 了解核心模式。
2. 在 `extensions` 中挑 1~2 个模式先落地。
3. 用 `session` 生命周期事件把监控/拦截能力串起来。

## 迁移建议

- 新项目：从「审计日志模式」「待办管理模式」开始。
- 安全优先项目：优先使用“危险命令拦截”“最小权限命令门禁”。
- 团队协作：配合 `git` 相关命令或事件做操作追踪。
